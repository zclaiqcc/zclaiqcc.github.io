培养计划

  • 第一天：申请权限和账号，新人介绍，熟悉团队成员，开发环境安装配置
  • 第一周：熟悉技术栈，熟悉团队文化，熟悉工作流
  • 第一月：尝试进行功能研发，在实际场景中快速成长/贡献
  • 后两月：加入项目组，参与到功能开发的工作流中，完成 Mentor 分配任务

第一周

  1. 配置开发环境
  2. 熟悉技术栈 - 暂无

  4. 小的 Demo 任务：Mentor 负责安排

第一月

  1. 参与部分 bug 修复
  2. 编写单元测试模块
  3. 熟悉技术栈 - 暂无
  4. 尝试参与功能开发：Mentor 负责安排

考核

考核周期

  • 第一月：研发部考核，Mentor 和技术负责人记录 - 暂无并打分
  • 第二月：项目组内考核，Mentor 和项目负责人记录 - 暂无并打分
  • 第三月：全公司考核，Mentor 和 CEO 记录 - 暂无并打分

考核内容

  1. 本月工作
  2. 遇到的困难和获得的成长
  3. 优缺点：停止、继续、开始
  4. 对团队的建议
  5. 提问环节
  6. Mentor 结语

日报制度

新人需要每天发送日报，Mentor 每日 review 日报并且回复

CodeReview

  • Mentor 负责安排清晰、明确、简单的开发任务
  • 每次设计方案和代码 MR/PR，Mentor 都必须认真 review，并给出反馈（好 or 具体改进建议）
  • 新人遇到问题要主动和 Mentor 沟通，Mentor 需要给与清晰的指导建议（提问原则：15min 没有任何头绪就问）

定期沟通

Mentor 需要主动每月和新人沟通至少一次，即时了解新人情况，给出建议，并有文档 - 暂无记录

Mentor 的职责

  1. 给新人制定培养计划，帮助新人熟悉项目业务
  2. 带领新人融入团队：包括熟悉团队成员，熟悉团队文化，衣食住行
  3. 每月至少需要和新人有一次深入的沟通，了解新人的当前情况，遇到的问题以及需要的帮助，并记录
  4. 新人培养过程中出现的一切事故，mentor 都需要负相应责任
  5. Mentor 随时拥有一票否决权
